# Workstreams

A workstream is a durable technical thread — architectural or multi-session in nature, checked in, and updated in-place as state changes. It is not a task log or a list of completed items.

Canonical rule: active workstreams live in `.ai/workstreams/`. Use `workstream-template.md` to start a new one.
